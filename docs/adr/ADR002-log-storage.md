# ADR002: Log Storage

### Status
Accepted

---

### Context
요청 단위로 정책 결정 결과를 디버깅하려면,
decision log를 조회하고 필터링하는 작업이 빈번하게 발생한다.

주요 사용 패턴:
- 특정 요청(request / decision) 단위 조회
- 시간 범위 기반 조회
- Deny 케이스 중심 조회
- 운영 중 수동 디버깅 목적의 조회

대규모 집계나 전문 검색보다는, 단건 또는 소수 건의 로그를 빠르게 확인하는 용도 중심

---

### Decision
초기 단계에서는 로그 저장소로 RDB(PostgreSQL)를 사용하는 것이 적절하다고 판단

- 요청 단위 조회에 유리하고
- 구조화된 필드 기반 필터링이 가능하며
- POC 단계에서 운영 및 관리 부담이 낮다

검색/집계 중심의 저장소(OpenSearch 등)는 추후 운영 목적이 명확해지는 시점에 검토

---

### Consequences
- 요청 단위 디버깅에 필요한 조회는 단순해진다.
- 스키마 변경에 대한 부담은 일부 존재한다.
- 대규모 집계나 통계 목적에는 한계가 있다.
